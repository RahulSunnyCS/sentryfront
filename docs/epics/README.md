# Epic Delivery Documents

One Markdown file per completed epic (or large collated chunk of work), named
`<epic-slug>.md` (kebab-case).

These are **human-facing delivery records** — what shipped, why it helps, the
limitations and tradeoffs and why we accepted them, the tests the AI ran, and
manual test cases anyone can run to verify the work by hand. They are **not** a
source of truth: the authoritative task state lives in `pipeline/tasks/*.json`,
`pipeline/progress.md`, and the root `TODO.md`.

## How files here are created

Generated by the **epic-doc-writer** agent
(`.claude/agents/epic-doc-writer.md`):

- Automatically in **Phase 7** (Final Review) of the pipeline, before the Final
  Summary Report.
- On demand via the **`/epic-doc`** command when a large chunk completes
  mid-pipeline (not only at the very end).

The agent writes only inside this directory, updates a file in place on re-run
(never forks a duplicate), and never edits the shared task ledger.
