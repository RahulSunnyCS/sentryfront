This document defines the required disclaimers that must appear in all VibeSafe security reports.

## Report Footer Disclaimer

**This text should appear at the bottom of every scan report (both web and PDF versions):**

This report was generated by VibeSafe, an automated security scanning tool. The scan was performed on **[DATE]** at **[TIME]** and analyzed publicly accessible content at the provided URL.

**Scan Type**: [Passive / Active]

---

### Limitations

#### 1. Not a Comprehensive Audit

This automated scan is NOT equivalent to:
- A manual penetration test conducted by security professionals
- A compliance audit (PCI-DSS, SOC 2, ISO 27001, HIPAA, etc.)
- A comprehensive security assessment of your entire infrastructure
- Legal or regulatory compliance certification

**Recommendation**: Engage qualified security consultants for formal audits.

#### 2. False Positives and False Negatives

- **False Positives**: Some findings may flag secure configurations as issues. Verify findings before remediation.
- **False Negatives**: Automated scans cannot detect all vulnerabilities. Absence of findings does not guarantee security.
- **Manual Review Required**: All findings should be reviewed by someone familiar with your application architecture.

#### 3. Evidence Redaction

- Sensitive data (API keys, secrets, tokens, cookies, passwords) is automatically redacted to `[first 4 chars]****[last 4 chars]`.
- Redaction is best-effort and may not catch all sensitive patterns.
- **Review reports carefully before sharing** to ensure no unredacted secrets remain.

#### 4. Severity Methodology

Severity ratings (CRITICAL, HIGH, MEDIUM, LOW, INFO) are assigned based on:
- Likelihood of exploitation
- Potential impact to confidentiality, integrity, and availability
- Industry best practices and OWASP guidelines

**Severity does not reflect business context.** A "HIGH" severity finding may be low-risk for your specific use case, and vice versa.

#### 5. Temporal Accuracy

- Findings reflect the state of the website **at the time of the scan**.
- Websites change frequently; findings may no longer be accurate.
- Re-scan regularly to monitor for new vulnerabilities.

---

### Grade Calculation

The overall security grade (A through F) is calculated as:

```
Score = (CRITICAL × 25) + (HIGH × 10) + (MEDIUM × 3) + (LOW × 1) + (INFO × 0)

A = 0–5 points
B = 6–20 points
C = 21–50 points
D = 51–90 points
F = 90+ points
```

**Grades are relative to scan findings only.** A grade of "A" means no critical issues were detected by this scan, NOT that your website is perfectly secure.

### AI-Generated Fix Prompts

Fix prompts labeled "AI Prompt" are generated by a large language model (Claude Sonnet via Anthropic API) and are provided for convenience. They are NOT:
- Guaranteed to be correct or complete
- Suitable for all application architectures
- A replacement for professional security remediation

**Always review AI-generated code** before deploying to production.

### UI Recommendations

- Use small, readable font (12pt for PDF, responsive web)
- Style as distinct section with border/background color
- Allow collapsing on web (default: collapsed, show "Read Disclaimers" link)
- For PDF, always include full text (cannot be hidden)

### Accessibility

- Use semantic HTML (`<section>`, `<h3>` headings)
- Ensure sufficient color contrast for disclaimers
- Make collapsible sections keyboard-navigable
