Priority findings

Page 04

These five findings move the most points on your grade. We recommend fixing them first; the remaining low-severity issues can be batched with normal maintenance.

High

Session cookies missing the Secure attribute

S3-02
📁 Security · Cookies 🎯 Set-Cookie: sid=… ⏱ Fix time ≈ 5 min

The session cookie set at login can be transmitted over plaintext HTTP. An attacker on the same Wi-Fi (coffee shop, conference, captive portal) can capture it and impersonate the user. HSTS mitigates this for modern browsers but doesn't protect first visits or non-browser clients.

Evidence
HTTP/2 200
set-cookie: sid=ey8nQ...; Path=/; HttpOnly; SameSite=Lax
                                          ↑ Secure flag missing
Remediation

Add Secure to every authentication cookie. In Next.js, set cookies().set('sid', token, { secure: true, httpOnly: true, sameSite: 'lax' }). If you use NextAuth, the cookie name prefix __Secure- enforces this at the browser level.

Medium

No Content-Security-Policy header

S2-01
📁 Security · Headers 🎯 All HTML responses ⏱ Fix time ≈ 30 min

Without a CSP, the browser will execute any script that ends up in the DOM — including injected ones from a stored-XSS payload or a compromised third-party tag. A CSP turns successful XSS attempts into harmless console warnings.

Evidence
curl -I https://example.com/ | grep -i csp
(no header returned)
Remediation

Start with a report-only policy to learn what your site loads: Content-Security-Policy-Report-Only: default-src 'self'; script-src 'self' 'unsafe-inline' https://js.stripe.com. After a week of clean reports, drop -Report-Only to enforce.

Medium

HSTS max-age is below 6 months

S2-04
📁 Security · TLS 🎯 Strict-Transport-Security ⏱ Fix time ≈ 5 min

You return max-age=86400 (1 day). HSTS only protects users for that long; if the browser hasn't visited your site in 24 hours, it can be downgraded to HTTP again. The HSTS preload list requires max-age ≥ 31536000.

Evidence
strict-transport-security: max-age=86400
Remediation

Raise to max-age=63072000; includeSubDomains; preload once you're confident every subdomain serves HTTPS. Submit at hstspreload.org.

Glossary & references

Page 11
CSP
Content Security Policy — an HTTP header that tells the browser which sources of scripts, styles, images, and connections are allowed. Mitigates XSS impact.
HSTS
HTTP Strict Transport Security — instructs browsers to refuse plaintext HTTP to your origin for a given duration. Preloading bakes this into the browser itself.
SRI
Subresource Integrity — a cryptographic hash on a <script> or <link> tag. The browser refuses to execute the resource if the bytes don't match.
SameSite
A cookie attribute (Strict, Lax, None) controlling whether the cookie is sent on cross-site requests. Mitigates CSRF.
LCP / INP / CLS
Core Web Vitals: Largest Contentful Paint (when the main content appears), Interaction to Next Paint (responsiveness), Cumulative Layout Shift (visual stability).
WCAG 2.2 AA
Web Content Accessibility Guidelines, level AA — the legal baseline for accessibility in the EU, UK, and most US public-sector procurement.
DAST
Dynamic Application Security Testing — probing a running application with real attack payloads, as opposed to static analysis of source code.
KEV / EPSS
CISA's Known Exploited Vulnerabilities catalog and FIRST's Exploit Prediction Scoring System — used to rank CVEs by real-world exploitation rather than CVSS alone.
